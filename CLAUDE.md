## 개발 환경 제약사항

### 사내/사외 네트워크 분리 구조

이 프로젝트는 **사내 서버에서 운영**되는 시스템이다.
Claude는 **사외 환경**에서만 접근 가능하므로, 실제 사내 DB(PostgreSQL, MongoDB, FTP 서버)에 직접 연결하여 검증할 수 없다.

### 원칙

- **테스트 / 개발 시**: 실제 DB 대신 **mockup 데이터**를 사용하여 구현하고 검증한다.
- **실제 배포 시**: 사내 DB(PostgreSQL, MongoDB, FTP)를 사용한다.
- Claude에게 코드 검증을 요청할 때는 mockup 기반으로 동작 확인을 진행한다.
- 실제 연결이 필요한 경우 사용자가 사내 환경에서 직접 실행하여 확인한다.

### 사내 DB 목록

| 용도 | 종류 | 비고 |
|------|------|------|
| 설비 마스터 | PostgreSQL | `eqp_info`, `sdwt_info` 테이블 |
| FTP 인증 정보 | MongoDB | `ADDCMP.FTP_STATUS` 컬렉션 |
| FTP 서버 | 각 설비 FTP | 직접 접근 불가 |
